Before I give the definition of a group, I want to introduce an example first.

Image transformations

Let's say you took some image, and you wanted to transform it. You open your favorite image editor, and you have some options: rotate the image however you want, or flip it across any axis.


You can press some sequence of these buttons, and the computer will apply these operations on the image.


Now, what if you had a looong sequence of operations? Performing a single image operation can already be quite expensive, let alone performing lots of them in quick succession.


We could always imagine what the image looks like after each step, but that seems time consuming. Given some sequence of operations, can you quickly determine what the final output image should be?

One thing we could do is composing the intermediate operations together. It means, we combine those operations into just a single one.
For example, suppose we wanted to rotate an image 90° clockwise after flipping it horizontally. Then that's the same as just flipping it over a diagonal!


These symbols are a bit hard to read, so instead I'll start using letters. Let $r$ denote a rotation 90° clockwise, and $s$ denote horizontal flip.
It turns out, we can represent every image operation in terms of just $r$ and $s$!


We use the symbol 1 for the "nothing" operation.

You can check for yourself:
A rotation 270° is really just three 90° rotations applied three times in a row.
A vertical flip is really just a 180° rotation followed by a horizontal flip!

One thing you might realize is that this representation is not unique. For example, while a diagonal flip is written $rrrs$, we can also do a flip then rotate, $sr$. So, $rrrs$ is actually the same as $sr$.


Let's go back to that looong sequence of operations I give earlier, and lets convert them to letters!


Now we have to figure out what $rrssrsrrrrrrsrrs$ means.
There are some things that could help us here:

If we have $ss$, then we're just doing two horizontal flips, making no change at all! In reality, $ss=1$.
If we have $rrrr$, then we're performing a 360° rotation, also the same as no change. $rrrr=1$.
As we saw before, we also have $rrrs=sr$.

We have our three "simplification rules": $$\begin{cases}ss=1\\rrrr=1\\rrrs=sr\end{cases}$$ Using this, we can simplify a decent chunk of our string:

As it turns out, the looong string $rrssrsrrrrrrsrrs$ is all just equal to $sr$, which is a single diagonal flip!


Basically, we've turned out problem about image transformations into one about algebraic manipulations.

It turns out, the set of all image transformations form a group under the composition operation.

What is a group?

When we say group, it consists of a set $X$ and a binary operation $\star$ on the set, which satisfies the following properties:
  1. For any elements $a,b,c\in X$, we have $(a\star b)\star c=a\star(b\star c)$.
  2. There is an identity element $e\in X$. It satisfies $a\star e=a$ and $e\star a=e$ for every $a\in X$.
  3. For every element $a\in X$, it has an inverse element $a^{-1}$. It satisfies $a\star a^{-1}=e$ and $a^{-1}\star a=e$.
If these properties are satisfied, we say that "$X$ is a group under the $\star$ operation". This group commonly notated as $(X,\star)$.

Here are two examples of groups which might be more familiar to you:

$(\mathbb Z,+)$ is a group

Let $\mathbb Z=\{\ldots,-3,-2,-1,0,1,2,3,\ldots\}$ be the set of all integers, and $+$ be the addition operation. Then $\mathbb Z$ is a group under addition because it has these properties:


Addition is associative: no matter how we group together the terms of a sum, we always get the same result!


Among all the integers, 0 is a very special: Whatever we try to add with 0, the number won't change! We could say that 0 preserved the identity of the original number.


Finally, every integer has its additive inverse, also called its inverse element. Whatever number we choose, there's always some number that can "cancel" it out to 0.

$(\mathbb Q^\times,\times)$ is a group

Let $\mathbb Q^\times$ be the set of all nonzero rational numbers. Anything of the form $\dfrac ab$ where $a,b$ are nonzero integers.
Let $\times$ be the multiplication operation. Then $\mathbb Q^\times$ is another example of a group:


Like before, multiplication is also associative.


This time, the special identity element is 1. Whatever we try to multiply with 1, the number won't change!


Every positive number has its reciprocal. When multiplied to the original number, it "cancels" out to our special number, 1! The reciprocal of a number is its inverse element.
Earlier, we also had an example of a group! This one is quite different because it is not commutative:

$D_8$ is a group

We defined some set of image transformations which form a group under composition:


This group is more commonly known as the dihedral group of order 8, denoted as $D_8$.

Intuitively, composing transformations is associative.


If we compose the first two, then compose with the third, the result is a 90° clockwise rotation.
If we instead do the first operation, then the composition of the other two, the result is still the same!

We can view composition as a sort of "shortcut" obtained from combining multiple operations together.

This group has an identity element. Can you guess what it is?


Yep, its just the transformation that does nothing.

Every single image transformation can be inverted.


If we rotate by 90° clockwise, we can apply a 90° counterclockwise rotation to put the image back to normal.
If we flipped horizontally, we can flip it again to put it back to normal.

Abelian and non-abelian groups

You might notice that the first two examples are commutative, while the last one is not.

A group is called abelian if the operation is commutative: $a\star b=b\star a$ for all $a,b\in G$.
If it is not commutative, the group is called non-abelian.

Non-examples

Now that we've seen three different examples, let me give some non-examples of groups.

$(\mathbb N,+)$ is NOT a group.

Let $\mathbb N=\{1,2,3,\ldots\}$ be the set of all positive integers, and $+$ be addition.

$\mathbb N$ does NOT form a group under addition because there is no identity element from that set.


0 would fit the equation, but it's not part of $\mathbb N$.

$(\mathbb Q,\times)$ is NOT a group .

Let $\mathbb Q$ be the set of all rational numbers, and $\times$ be multiplication.

Although 1 is an identity element, $\mathbb Q$ does NOT form a group under $\times$ because 0 has no inverse element from that set.


This is why in our example earlier, 0 had to be removed in $\mathbb Q^\times$.

Abelian groups

These are groups where the operation is commutative.

Modulo arithmetic

Let $C_n=\{0,1,2,\ldots,n-1\}$. In this group, the operation is adding two numbers $a+b$ modulo $n$.


This is also known as the cyclic group of order $n$.

Instead of $C_n$, most people would also denote it by $\mathbb Z/n\mathbb Z$.

Unit circle

Let $S^1=\{z\in\mathbb C:|z|=1\}$ be the set of all complex numbers on the unit circle, and $\times$ be multiplication.

You should verify that for any $z_1,z_2\in S^1$, it is true that $z_1z_2\in S^1$.

$(S^1,\times)$ forms the circle group.

Trivial group

This group only has one element: $\{1\}$, and an operation $\cdot$ where $1\cdot1=1$.

It is the trivial group.

Non-abelian groups

These are groups where the operation is not commutative.

Symmetric group

Let $S_n$ be the set of all permutations on $n$ objects.
$S_n$ forms a group under the composition operation.


This is also known as the symmetric group of order $n$.

Rubik's Cube

Let $G$ be the set of moves you could do on a Rubik's Cube, and $\cdot$ be the composition of cube moves.

Then, $(G,\cdot)$ forms the Rubiks Cube group.

Matrix multiplication

Let $M_n$ be the set of all invertible $n\times n$ matrices.
$M_n$ forms a group under matrix multiplication. $$\begin{pmatrix}0&1\\2&3\end{pmatrix}\cdot\begin{pmatrix}4&-5\\1&2\end{pmatrix}=\begin{pmatrix}1&2\\11&-4\end{pmatrix}$$ It has the fancy name of general linear group of degree $n$.

Product group

Another important example comes from combining groups together, like a Frankenstein monster.

Let $(G,\star)$ and $(H,*)$ be two groups. Then a product group $(G\times H,\cdot)$ is defined as follows:

The elements of the group are ordered pairs $(g,h)\in G\times H$. The group operation is $$(g_1,h_1)\cdot(g_2,h_2)=(g_1\star g_2,h_1\star h_2)$$

Isomorphism

Sometimes, we'll encounter two different groups which happen to "behave exactly the same way". Here are some examples: :todo:explain
  1. even integers <-> integers
  2. mod 12 is isomorphic to mod 3 * mod 4
  3. complex unit circle, reals mod 1, are all isomorphic
  4. addition in reals <-> multiplication in positive reals
  5. primitive roots modulo
If two groups happen to behave in exactly the same way, we say that these two groups are isomorphic to each other! The elements behave exactly the same, and the only difference between these two groups is the labelling of these elements.

Okay, but how do we rigorize this notion of "isomorphism"? Well, we only need to look what the group operation does to these elements! If a relabelling of elements exists between the two groups, such that the underlying "structure" of that operation is preserved, then we know they are isomorphic.

An isomorphism between two groups $(G,\star)$ and $(H,*)$ is defined to be a bijection $\phi:G\to H$ satisfying $$\phi(g_1\star g_2)=\phi(g_1)*\phi(g_2)\quad\text{for all }g_1,g_2\in G$$ Essentially, this is just a fancy way of saying that $\phi$ is a "relabeling" of elements in $G$ to be those of $H$, so that every interaction between elements are preserved.
